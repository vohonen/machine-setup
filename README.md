# machine-setup

Cross-platform (macOS + Ubuntu) machine provisioning. One OS-detecting bootstrap, idempotent,
dotfiles symlinked into `$HOME` via GNU stow so editing a config in `$HOME` edits this repo directly.

## Bootstrap a new machine

```sh
git clone --recurse-submodules git@github.com:vohonen/machine-setup.git
cd machine-setup
./bootstrap.sh        # -> install/macos.sh on a Mac, install/linux.sh on Ubuntu
```

`--recurse-submodules` pulls `vohonen/nvim-config` into `stow/common/.config/nvim`.

## Layout

```
bootstrap.sh            OS detect -> install/macos.sh | install/linux.sh
Brewfile                macOS packages (brew bundle)
apt-packages.txt        Ubuntu packages (apt)
install/macos.sh        brew bundle, stow, submodule, keylayout, defaults, git identity
install/linux.sh        apt, stow, submodule, xkb keyboard, git identity
stow/common/.config/nvim  git submodule -> vohonen/nvim-config
stow/macos/             .zshrc .zprofile .zsh_aliases .config/aerospace
stow/linux/             .bashrc .bash_aliases
macos/                  Finnish-TeX.keylayout, gen_keylayout.py, macos-defaults.sh
linux/                  fi (xkb), setup-keyboard.sh
```

## Manual steps after `./bootstrap.sh`

The installer prints these (they need a password or a GUI):

- **SSH -> GitHub:** `ssh-keygen -t ed25519 -C "vili.kohonen@protonmail.com"`, add `~/.ssh/config`
  (Host github.com / AddKeysToAgent yes / UseKeychain yes / IdentityFile ~/.ssh/id_ed25519),
  `ssh-add --apple-use-keychain ~/.ssh/id_ed25519`, paste `~/.ssh/id_ed25519.pub` into GitHub.
- **macOS keyboard:** Settings > Keyboard > Input Sources: add **Finnish TeX**, set primary (log out/in).
  Settings > Keyboard > Keyboard Shortcuts > Modifier Keys: Caps Lock -> Escape.
- **AeroSpace:** grant Accessibility permission on first launch.
- **Dropbox + Full Disk Access:** modern Dropbox syncs to `~/Library/CloudStorage/Dropbox`
  (with a `~/Dropbox` symlink to it). macOS blocks terminal apps from that File Provider path
  until they have Full Disk Access — add **Alacritty** under Settings > Privacy & Security >
  Full Disk Access, then **fully quit & relaunch** it (nvim launched from it inherits the grant).
  Symptom without it: `ls ~/Dropbox` → "Operation not permitted" (the files are fine; it's TCC).
- **Skim (VimTeX inverse search):** Skim > Settings > Sync: enable "Check for file changes";
  set Preset = Custom, Command = `nvim`,
  Arguments = `--headless -c "VimtexInverseSearch %line '%file'"`. (Forward search needs no config —
  VimTeX calls Skim's `displayline`.)
- **Mail:** Settings > Internet Accounts > Google -> enable Mail; Mail > Settings > General > default reader.

## Editing & syncing configs

Stow symlinks the dotfiles, so **editing the file in `$HOME` is editing this repo** — no copy step:

```sh
# edit ~/.zshrc, ~/.config/aerospace/aerospace.toml, etc. in place, then:
cd ~/machine-setup && git commit -am "tweak" && git push
# on another machine:  cd ~/machine-setup && git pull   (symlinks already point here)
```

**nvim is the exception** — it's a submodule with its own remote. Edit under `~/.config/nvim`, then:

```sh
cd ~/machine-setup/stow/common/.config/nvim && git commit -am "..." && git push
cd ~/machine-setup && git commit -am "bump nvim-config" && git push   # record new pointer
```

Pull others' nvim changes: `git -C ~/machine-setup submodule update --remote --merge`.

## Keyboard layout (macOS)

`macos/Finnish-TeX.keylayout` is generated by `macos/gen_keylayout.py` from the Ubuntu `fi` kotoistus
block: `$ \ ^ ~ { }` at base/shift, Finnish `å ä ö` and brackets via Option. Regenerate after edits:
`python3 macos/gen_keylayout.py`, reinstall, log out/in. Two ISO keys (codes 50 = left-of-1 -> å,
10 = left-of-Z -> `< > |`) may need swapping in the generator if they land wrong on UK Apple ISO.
